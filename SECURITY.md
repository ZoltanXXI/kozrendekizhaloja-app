# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it **privately** by sending an email to:

**turo.zoli99@gmail.com**  

Do not open an issue in the public repository for vulnerabilities, as this may expose users to risk.

Please include:

- A clear description of the vulnerability
- Steps to reproduce it
- The environment in which it occurs
- Potential impact

We aim to respond to security reports within **48 hours**.

---

## Supported Versions

We actively maintain and support the following versions:

- `main` branch (latest stable)
- Release tags: `v1.x`, `v2.x`

If your vulnerability affects an unsupported version, we may not provide a fix, but we will acknowledge the report.

---

## Security Practices

- Keep your dependencies up-to-date
- Avoid exposing `.env` files or API keys
- Use HTTPS and other best practices for secure communication

---

## Thank you

We take security seriously and appreciate responsible disclosure. Your help keeps the project safe for everyone.
